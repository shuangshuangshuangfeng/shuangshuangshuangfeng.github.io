Java集合框架介绍

Java的集合框架有下图所示：

Collection 有序接口

Collection的主要方法有:

  • boolean add(Object o)添加对象到集合
  • boolean remove(Object o)删除指定对象
  • int size() 返回当前集合中的元素
  • boolean contains(Object O)查找集合中是否有指定对象
  • boolean isEmpty() 判断集合是否为空
  • Iterator iterator() 返回一个迭代器
  • boolean containsAll(Collection c) 查找集合中是否含有集合c的元素
  • boolean addAll(Collection c) 将集合c中的元素添加至集合中
  • void clear() 删除集合中的所有元素
  • void removeAll(Collection c)从集合中删除c集合中也有的元素
  • void retainAll(Collection c)从集合中删除集合c中不包含的元素

List抽象接口

List抽象接口可以重复有序

List的主要方法

  • void add(int index, Object element)在指定位置添加一个元素
  • boolean addAll(int index, Collect c)将集合c中的元素添加到指定的位置
  • Object get(int index) 返回List中某个指定位置的元素
  • int indexOf(Object o) 返回第一个出现元素o的位置
  • Object remove(int index)删除指定位置的元素
  • Object set(int index, Object element)用元素element取代位置为index上的元素，返回被取代的元素
  • void sort() 排序

List主要接口对象

LinkedList时基于动态数组的集合类，ArrayList是基于链表的集合类。

  • LinkedList没有同步方法
  • ArrayList是非同步的; 扩容增量时原容量的0.5倍+1,如：ArrrayList的容量为10，一次扩容后的容量时10/2*3+1=16
  • Vector是同步的(synchronized)，类似于ArrayList; Vector的加载因子是1，即：当元素个数超过容量长度时，进行扩容，扩容增量为原来的一倍。

ArrayList和Vector的初始化长度是10

Set集合

Java中使用Set接口的集合不允许有重复值,也就是说Set中的每一个元素都是唯一的。
因为Set是一个抽象接口，所以不能直接调用。
Set set = new Set()这样是不正确的。

Set的主要方法

  • int size() 获取集合中元素的个数
  • add(Object o)向集合中添加元素
  • Object remove(Object o) 在集合中删除元素o
  • boolean contains(Object o) 判断集合中是否含有元素o
  • iterator() 排序

Set接口的实现类-HashSet

HashSet通过Hash算法排布集合内的元素，是一个无序，不可重复的集合。

  • HashSet不能保证元素的顺序，不可重复，不是线程安全的，集合元素可以为Null.
  • 其底层实现实际上是一个数组，存在的意义就是加快查询速度.
  • 对于HashSet集合中存在两个对象通过equals()方法返回true,那么这两个对象的hashCode值也应该相同。
  • HashSet的初始化长度是16
  • 加载因子为0.75，即: 元素个数超过容量长度的0.75倍时，进行扩容
  • 扩容增量是原容量的1倍， 如：HashSet的容量是16，一次扩容后是32.
    注意: 每一个存储到哈希表中的对象，都要提供hashCode()和equals()方法的实现，用来判断是否是一个对象。

    TreeSet

    Map

    Map接口的实现类有：HashMap, Hashtable，TreeMap。

HashMap

最常用的Map，它根据键的HashCode值存储数据，根据键可以直接获取它的值，具有很快的访问速度.

  • HashMap最多只允许一条记录的键为Null(多条会覆盖);
  • 允许多条记录的值为 Null;
  • 非同步的。
  • HashMap的初始化长度是16，是为了降低碰撞的几率；
  • 扩容增量是原来的1倍
    加载因子为0.75，即：当元素个数超过容量的0.75倍时进行扩容

HashMap的扩容:
默认情况下，数组大小为16，那么当hashmap中元素个数超过160.75=12的时候，就把数组的大小扩展为216=32，即扩大一倍，然后重新计算每个元素在数组中的位置，而这是一个非常消耗性能的操作，所以如果我们已经预知hashmap中元素的个数，那么预设元素的个数能够有效的提高hashmap的性能。

HashMap以自定义对象作为key有什么要求?
用自定义类作为key，必须重写equals()和hashCode()方法。
自定义类中的equals() 和 hashCode()都继承自Object类。
Object类的hashCode()方法返回这个对象存储的内存地址的编号。
而equals()比较的是内存地址是否相等。

因为HashMap执行get()方法的时候，执行了equals()方法和hashCode()方法，在扩增的时候会new一个新的对象时，地址变了，不能保证hash值和equals()及如果还是一样，所以取不到对应的value。

LinkedHashMap

用法跟HashMap基本一致，它是基于链表和哈希表结构的所以具有存取有序，键不重复的特性
存的顺序和遍历出来的顺序是一致的

Hashtable

  • 与HashMap类似，不同的是key和value的值均不允许为null，
  • 支持线程的同步.即任一时刻只有一个线程能写Hashtable,因此也导致了Hashtale在写入时会比较慢。

    TreeMap

  • 能够把它保存的记录根据键(key)排序,默认是按升序排序，也可以指定排序的比较器，当用Iterator 遍历TreeMap时，得到的记录是排过序的。
  • TreeMap不允许key的值为null。
  • 非同步的。

    有关集合的面试题

  1. List初始化大小:
    List arrayList = new ArrayList();
    如果使用默认的构造方法，初始容量被设置为10，当ArrayList中的元素超过10个的时候，会使数组的大小增长到16.
    通过调试可以看的到动态增长的数量变化是: 10->16->25->38->58->88->…
    增长的计算公式为: 新容量 = (旧容量*3)/2+1

  2. ArrayList和Vector(旧的)的区别:
    共同点: 这两个类都实现了List接口，他们都是有序集合，即存储在这两个集合中的元素的位置都是有顺序的，相当于一种动态的数组，我们以后可以按位置索引号取出某个元素，并且其中的数据是允许重复。
    区别:

  • 同步性: Vector是线程安全的，也就是说他们的方法之间是线程同步的，而ArrayList是线程不安全的，它的方法之间是线程不同步的。如果只有一个线程会访问到集合，那最好是使用ArrayList，因为它不考虑线程安全，效率会高些；如果有多个线程会访问到集合，那最好是使用Vector，因为不需要我们自己再去考虑和编写线程安全的代码。
  • 数据增长:ArrayList与Vector都有一个初始的容量大小，当存储进他们里面的元素超过了容量的时候,就需要增加ArrayList与Vector的存储空间。Vector默认增长为原来两倍，而ArrayList的增长策略在文档中没有明确规定（从源代码看到的是增长为原来的1.5倍）。ArrayList与Vector都可以设置初始的空间大小，Vector还可以设置增长的空间大小，而ArrayList没有提供设置增长空间的方法。
  1. HashMap与Hashtable的区别:
  • Hashtable是基于陈旧的Dictionary类， HashMap是Java 1.2引进的Map接口的一个实例。
  • Hashtable是同步的，是线程安全的。HashMap是不同步的，非线程安全。
  • HashMap可以允许空值，Hashtable不允许空值。

HashMap的实现原理

HashCode

  • hashCode的存在主要是用于查找的快捷性，如Hashtable，HashMap等，hashCode是用来在散列存储结构（也就是hash表）中确定对象的存储位置。
  • 如果两个对象相同，就是适用于equals()方法，那么这两个对象的hashCode一定要相同。
  • 如果对象的equals()方法被重写，那么对象的hashCode也尽量重写，并且产生hashCode使用的对象，一定要和equals()方法中使用的一致，否则将违反上面提到的第二点，在Set集合中会出现问题。
  • 两个对象的hashCode相同，并不一定表示两个对象就相同，也就是不一定适用与equals()方法，者能够说明这两个对象在散列存储结构中，如Hashtable，他们是存放在一个’链子’的。

HashMap的构成

HashMap实际上是一种‘散列链表’的数据结构，即数组和链表的结合体。
散列链表又叫Hash表

实际上HashMap底层就是一个Hash表的数组结构，数组中的每一项又是一个链表，当新建一个HashMap的时候，就会初始化一个数组。

CurrentHashMap

ConcurrentHashMap比HashMap多出了一个类Segment，而Segment是一个可重入锁。ConcurrentHashMap是使用了锁分段技术来保证线程安全的。
锁分段计数:首先将数据分成一段一段的存储，然后给每一段数据配一把锁，当一个线程占用锁访问其中一个段数据的时候，其他段的数据也能被其他线程访问。
ConcurrentHashMap提供了与Hashtable和SynchronizedMap不同的锁机制。Hashtable中采用的锁机制是一次锁住整个hash表，从而在同一时刻只能由一个线程对其进行操作；而ConcurrentHashMap中则是一次锁住一个桶。

ConcurrentHashMap默认将hash表分为16个桶，诸如get、put、remove等常用操作只锁住当前需要用到的桶。这样，原来只能一个线程进入，现在却能同时有16个写线程执行，并发性能的提升是显而易见的。
